# 剧本杀司机接单台 - 启动与验收说明

## 🚀 快速启动（二选一）

### 方式一：桌面客户端模式（推荐验收使用）

```bash
# 进入项目目录
cd d:\trae-bz\TraeProjects\2125

# 启动桌面客户端（独立窗口）
npm start
# 或
npm run dev:electron
```

**说明：**
- 首次启动会自动下载 Electron 二进制文件（约 100MB），请耐心等待
- 下载完成后会自动弹出独立的桌面窗口，标题为「剧本杀司机接单台」
- 窗口默认大小 1280×800，最小尺寸 960×640
- 无需打开浏览器，应用完全独立运行

---

### 方式二：浏览器开发模式

```bash
# 启动 Vite 开发服务器
npm run dev
```

然后在浏览器打开：http://localhost:5175/

## ✅ 验收步骤

### 第 1 步：启动桌面客户端
1. 打开命令行，进入项目目录 `d:\trae-bz\TraeProjects\2125`
2. 执行命令 `npm start`
3. 等待下载和启动完成（首次约需 1-3 分钟）
4. 确认看到独立桌面窗口，窗口标题显示「剧本杀司机接单台」

### 第 2 步：验证报价功能和持久化
1. 在今日列表中找到一个「待报价」状态的订单，点击卡片
2. 进入详情后点击「立即报价」按钮
3. 填写表单并提交（可乘人数 > 0，报价 > 0）
4. 提交后确认：
   - 详情底部状态变为「已提交报价，等待门店确认...」
   - 详情中出现绿色的「我的报价」卡片，显示完整的车型、可乘人数、报价、到达时间
   - 关闭侧边栏，列表中对应卡片显示「已报价 ¥xxx」标签
5. **验证持久化**：
   - 按 F5 刷新窗口，或关闭窗口后重新执行 `npm start`
   - 打开刚才报价的订单详情，确认「我的报价」信息依然存在
   - 列表中状态依然显示「已报价」

### 第 3 步：验证服务完成功能
1. 找到状态为「已接单」或「服务中」的订单（今日列表第 2、4 条）
2. 点击进入详情，点击底部「确认完成服务」按钮
3. 填写：
   - 实际出发时间（早于结束时间）
   - 实际结束时间
   - 选择 1-2 个服务标签
   - 填写备注说明
4. 提交后确认：
   - 详情中出现绿色的「服务完成记录」卡片，包含实际时间、标签、备注
   - 列表状态变为「已完成」灰色标签
5. **验证持久化**：
   - 刷新或重启应用，确认完成记录依然存在

### 第 4 步：验证列表和详情状态一致性
1. 切换「今日」和「明日」标签，确认每个订单的状态标签和进入详情后一致
2. 切换各种筛选条件（离我近、夜间单、大车优先），再切回「全部」，确认状态没有变化
3. 对一个订单报价后，切换今日/明日标签再切回来，确认列表和详情状态一致

### 第 5 步：验证表单有效性检查
1. 打开一个「待报价」订单，点击「立即报价」
2. 将可乘人数改为 0 或负数，点击提交，确认提示错误，数据不保存
3. 将报价改为 0 或清空，点击提交，确认提示错误
4. 将到达时间清空，点击提交，确认提示错误
5. 修复错误后重新提交，确认成功保存且详情中没有 NaN 等异常显示

## 💡 常见问题

### Q: 首次启动 npm start 很久没反应？
A: 正在下载 Electron 二进制文件（约 100MB），取决于网速通常需要 1-5 分钟。看到 Vite 启动后继续等待即可。

### Q: 如何确认是桌面窗口而不是浏览器？
A: 检查窗口标题栏，独立桌面窗口会有「剧本杀司机接单台」标题和最小化/最大化/关闭按钮，没有浏览器的地址栏。

### Q: 报价后状态没更新？
A: 请先刷新页面或关闭重开。如果还是旧状态，可以点击页面右上角「重置数据」按钮恢复初始数据后重试。
